로컬 독립형 Pulsar 구성

로컬 개발 및 테스트를 위해 여러분의 컴퓨터에서 Pulsar를 독립형 모드로 실행할 수 있습니다. The standalone mode includes a Pulsar broker, the necessary RocksDB and BookKeeper components running inside of a single Java Virtual Machine (JVM) process.

Pulsar in production?
If you're looking to run a full production Pulsar installation, see the Deploying a Pulsar instance guide.

독립형 Pulsar 설치

This tutorial guides you through every step of installing Pulsar locally.

System requirements

Currently, Pulsar is available for 64-bit macOS, Linux, and Windows. To use Pulsar, you need to install 64-bit JRE/JDK 8 or later versions.

Tip
By default, Pulsar allocates 2G JVM heap memory to start. It can be changed in conf/pulsar_env.sh file under PULSAR_MEM. This is extra options passed into JVM.

Note
Broker is only supported on 64-bit JVM.

Binary release를 이용한 Pulsar 설치

Pulsar를 시작하려면 다음 방법 중 하나로 바이너리 tarball 릴리스를 다운로드하십시오.

  • apache mirror 에서 다운로드 (Pulsar 2.9.1 binary release)

  • pulsar 다운로드 페이지에서 다운로드

  • pulsar 릴리스 페이지에서 다운로드

  • wget 사용:

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.9.1/apache-pulsar-2.9.1-bin.tar.gz

Tarball을 다운로드 한 후 tarball을 풀고 cd 명령을 사용하여 결과 디렉토리로 이동하십시오.

$ tar xvfz apache-pulsar-2.9.1-bin.tar.gz
$ cd apache-pulsar-2.9.1

What your package contains

The Pulsar binary package initially contains the following directories:

디렉토리포함
binPulsar's command-line tools, such as pulsar and pulsar-admin.
confConfiguration files for Pulsar, including broker configuration and more.
Note: Pulsar standalone uses RocksDB as the local metadata store and its configuration file path metadataStoreConfigPath is configurable in the standalone.conf file. For more information about the configurations of RocksDB, see here and related documentation.
examplesPulsar Functions 예제가 포함된 Java Jar 파일들이 있습니다.
인스턴스Pulsar Functions을 위하여 생성된 아티펙트.
libThe JAR files used by Pulsar.
licensesPulsar 코드베이스의 여러 구성 요소에 대한 .txt형식의 오픈 소스 라이센스 파일이 있습니다.

Pulsar를 실행하면 이 디렉토리들이 생성됩니다.

디렉토리포함
데이터The data storage directory used by RocksDB and BookKeeper.
로그Logs created by the installation.

Tip
If you want to use builtin connectors and tiered storage offloaders, you can install them according to the following instructions：

  • 빌트인 커넥터 설치 (optional)
  • 계층형 스토리지 오프로더 설치 (optional)

Otherwise, skip this step and perform the next step Start Pulsar standalone. Pulsar can be successfully installed without installing bulitin connectors and tiered storage offloaders.

빌트인 커넥터 설치 (optional)

Pulsar는 2.1.0-incubating 릴리즈 이후, 모든 builtin 커넥터를 포함하는 별도의 바이너리 배포를 릴리즈합니다. 이러한 빌트인 커넥터를 활성화 하려면, 다음 방법 중 하나로 커넥터 tarball 릴리즈를 다운로드 할 수 있습니다.

  • apache 미러 Pulsar IO Connectors 2.9.1 release

  • pulsar 다운로드 페이지에서 다운로드

  • pulsar 릴리스 페이지에서 다운로드

  • wget 사용:

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.9.1/connectors/{connector}-2.9.1.nar

nar 파일을 다운로드 한 후, pulsar 디렉토리의 connectors 디렉토리로 복사하십시오. 예를 들어, pulsar-io-aerospike-{{pulsar: version}}.nar 커넥터 파일을 다운로드 한 경우 다음 명령을 입력하십시오:

$ mkdir connectors
$ mv pulsar-io-aerospike-2.9.1.nar connectors

$ ls connectors
pulsar-io-aerospike-2.9.1.nar
...

Note

  • If you are running Pulsar in a bare metal cluster, make sure connectors tarball is unzipped in every pulsar directory of the broker (or in every pulsar directory of function-worker if you are running a separate worker cluster for Pulsar Functions).

  • If you are running Pulsar in Docker or deploying Pulsar using a docker image (e.g. K8S or DC/OS, you can use the apachepulsar/pulsar-all image instead of the apachepulsar/pulsar image. apachepulsar/pulsar-all image has already bundled all builtin connectors.

계층형 스토리지 오프로더 설치 (optional)

Tip

  • Since 2.2.0 release, Pulsar releases a separate binary distribution, containing the tiered storage offloaders. - To enable tiered storage feature, follow the instructions below; otherwise skip this section.

계층적 스토리지 오프로더를 시작하려면, 다음중 한 방법으로 모든 브로커 노드에 오프로더 tarball 릴리즈를 다운로드 해야합니다:

  • Apache 미러 Pulsar 계층적 스토리지 오프로더2.9.1 release

  • pulsar 다운로드 페이지에서 다운로드

  • pulsar 릴리스 페이지에서 다운로드

  • wget 사용:

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.9.1/apache-pulsar-offloaders-2.9.1-bin.tar.gz

tarball을 다운로드 한 후, 오프로더 패키지를 untar하고 오프로더를 Pulsar 디렉토리의 offloaders로 복사하십시오:

$ tar xvfz apache-pulsar-offloaders-2.9.1-bin.tar.gz

// Pulsar 디렉토리 내부에 `apache-pulsar-offloaders-2.9.1`라는 디렉토리를 찾습니다.
// 그런다음 offloaders 를 복사합니다.

$ mv apache-pulsar-offloaders-2.9.1/offloaders offloaders

$ ls offloaders
tiered-storage-jcloud-2.9.1.nar

계층적 저장소 구성에 관한 자세한 정보는 Tiered storage cookbook를 참고하십시오.

Note

  • If you are running Pulsar in a bare metal cluster, make sure that offloaders tarball is unzipped in every broker's pulsar directory.

  • If you are running Pulsar in Docker or deploying Pulsar using a docker image (e.g. K8S or DC/OS), you can use the apachepulsar/pulsar-all image instead of the apachepulsar/pulsar image. apachepulsar/pulsar-all image has already bundled tiered storage offloaders.

Pulsar 독립형 모드 시작

최신 로컬 사본 릴리즈를 가지고 있다면, bin 디렉토리에 있는 pulsar 명령어를 사용하여 독립형 모드의 Pulsar 로컬 클러스터를 시작 할 수 있습니다.

$ bin/pulsar standalone

Pulsar를 성공적으로 시작한 경우 다음과 같이 INFO 레벨의 로그 메시지가 표시됩니다.

21:59:29.327 [DLM-/stream/storage-OrderedScheduler-3-0] INFO  org.apache.bookkeeper.stream.storage.impl.sc.StorageContainerImpl - Successfully started storage container (0).
21:59:34.576 [main] INFO  org.apache.pulsar.broker.authentication.AuthenticationService - Authentication is disabled
21:59:34.576 [main] INFO  org.apache.pulsar.websocket.WebSocketService - Pulsar WebSocket Service started

Tip

  • The service is running on your terminal, which is under your direct control. If you need to run other commands, open a new terminal window.
    You can also run the service as a background process using the bin/pulsar-daemon start standalone command. For more information, see pulsar-daemon.

  • By default, there is no encryption, authentication, or authorization configured. Apache Pulsar는 원격 서버에서 특별한 권한 없이도 억세스 할 수 있습니다. 배포에 보안을 적용하려면 보안 개요 문서를 확인하십시오.

  • When you start a local standalone cluster, a public/default namespace is created automatically. namespace는 개발 목적으로 사용됩니다. All Pulsar topics are managed within namespaces. 자세한 내용은 Topics를 참조하십시오.

독립형 Pulsar 사용

Pulsar는 pulsar-client라는 CLI 도구를 제공합니다. pulsar-client 를 사용하면 실행중인 클러스터에서 Pulsar Topic 메시지를 소비하고 생성 할 수 있습니다.

메시지 소비

다음 명령은 구독 이름이 first-subscription인 my-topic Topic의 메시지를 컨슘합니다.

$ bin/pulsar-client consume my-topic -s "first-subscription"

메시지가 성공적으로 소비 된 경우, pulsar-client 로그에 다음과 같은 확인 메시지가 표시됩니다:

22:17:16.781 [main] INFO  org.apache.pulsar.client.cli.PulsarClientTool - 1 messages successfully consumed

Tip
As you have noticed that we do not explicitly create the my-topic topic, to which we consume the message. When you consume a message to a topic that does not yet exist, Pulsar creates that topic for you automatically. Producing a message to a topic that does not exist will automatically create that topic for you as well.

메시지 생산

다음 명령은 my-topic topic에 hello-pulsar라는 메시지를 프로듀스 합니다.

$ bin/pulsar-client produce my-topic --messages "hello-pulsar"

메시지가 topic에 성공적으로 게시 된 경우, pulsar-client 로그에 다음과 같은 확인 메시지가 표시됩니다:

22:21:08.693 [main] INFO  org.apache.pulsar.client.cli.PulsarClientTool - 1 messages successfully produced

독립형 Pulsar 중지

로컬 독립형 Pulsar 중지를 하려면 Ctrl+C를 누르십시오.

Tip
If the service runs as a background process using the bin/pulsar-daemon start standalone command, then use the bin/pulsar-daemon stop standalone command to stop the service.

For more information, see pulsar-daemon.
