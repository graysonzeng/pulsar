Docker 에서 독립 모드 Pulsar 구성

For local development and testing, you can run Pulsar in standalone mode on your own machine within a Docker container.

If you have not installed Docker, download the Community edition and follow the instructions for your OS.

Docker에서 Pulsar 시작하기

  • MacOS, Linux 및 Windows의 경우:

    $ docker run -it -p 6650:6650  -p 8080:8080 --mount source=pulsardata,target=/pulsar/data --mount source=pulsarconf,target=/pulsar/conf apachepulsar/pulsar:2.9.1 bin/pulsar standalone

이 명령에 대해 몇 가지 참고 할 사항: * 컨테이너를 다시 시작할 때마다 "새로" 시작하지 않도록, data와 metadata 및 configuration이 Docker 볼륨에 유지되도록 합니다. For details on the volumes you can use docker volume inspect <sourcename> * For Docker on Windows make sure to configure it to use Linux containers * The docker container will run as UID 10000 and GID 0, by default. You'll need to ensure the mounted volumes give write permission to either UID 10000 or GID 0. Note that UID 10000 is arbitrary, so it is recommended to make these mounts writable for the root group (GID 0).

Pulsar가 성공적으로 시작하면, 다음과 같은 INFOlevel 로그 메시지가 표시됩니다.

08:18:30.970 [main] INFO  org.apache.pulsar.broker.web.WebService - HTTP Service started at http://0.0.0.0:8080
...
07:53:37.322 [main] INFO  org.apache.pulsar.broker.PulsarService - messaging service is ready, bootstrap service port = 8080, broker url= pulsar://localhost:6650, cluster=standalone, configs=org.apache.pulsar.broker.ServiceConfiguration@98b63c1
...

Tip
When you start a local standalone cluster, a public/default namespace is created automatically. namespace는 개발 목적으로 사용됩니다. All Pulsar topics are managed within namespaces. 자세한 내용은 Topics를 참조하십시오.

Docker에서 Pulsar 사용

Pulsar는 Java, Go, Python 및 C++용 클라이언트 라이브러리를 제공합니다. 로컬 독립형 클러스터를 실행중인 경우, 다음 root URL 중 하나를 사용하여 클러스터와 상호 작용할 수 있습니다:

  • pulsar://localhost:6650
  • http://localhost:8080

The following example will guide you get started with Pulsar quickly by using the Python client API client API.

PyPI에서 직접 Pulsar Python 클라이언트 라이브러리를 설치하십시오:

$ pip install pulsar-client

메시지 소비

Consumer를 작성하고 topic을 구독하십시오.

import pulsar

client = pulsar.Client('pulsar://localhost:6650')
consumer = client.subscribe('my-topic',
                            subscription_name='my-sub')

while True:
    msg = consumer.receive()
    print("Received message: '%s'" % msg.data())
    consumer.acknowledge(msg)

client.close()

메시지 생산

이제 Producer를 시작하여 테스트 메시지를 보내십시오:

import pulsar

client = pulsar.Client('pulsar://localhost:6650')
producer = client.create_producer('my-topic')

for i in range(10):
    producer.send(('hello-pulsar-%d' % i).encode('utf-8'))

client.close()

Topic 통계 얻기

Pulsar에서는 REST, Java 또는 명령 행 도구를 사용하여 시스템의 모든 측면을 제어 할 수 있습니다. API에 대한 자세한 내용은 Admin API Overview를 참조하십시오:

In the simplest example, you can use curl to probe the stats for a particular topic:

$ curl http://localhost:8080/admin/v2/persistent/public/default/my-topic/stats | python -m json.tool
