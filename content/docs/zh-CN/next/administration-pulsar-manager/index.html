Pulsar Manager

Pulsar Manager 是一个网页式可视化管理与监测工具，支持多环境下的动态配置。可用于管理和监测租户、命名空间、topic、订阅、broker、集群等。

提示：如果你正在使用Pulsar dashboard 监控Pulsar当前的状态，我们建议你使用Pulsar Manager 替代。 Pulsar dashboard 已弃用。

安装

快速安装

在 Docker 容器中运行是使用 Pulsar Manager 最简单的方法。

docker pull apachepulsar/pulsar-manager:v0.2.0
docker run -it \
    -p 9527:9527 -p 7750:7750 \
    -e SPRING_CONFIGURATION_FILE=/pulsar-manager/pulsar-manager/application.properties \
    apachepulsar/pulsar-manager:v0.2.0
  • Pulsar Manager分为前端和后端服务，前端的服务端口为9527，后端服务端口为7750。
  • SPRING_CONFIGURATION_FILE: Spring 默认使用的配置文件
  • Pulsar Manager默认使用HerdDB作为数据库， HerdDB是一个基于Java实现的SQL分布式数据库，你可以在 herddb.org找到更多相关信息。

配置数据库和JWT身份认证

配置数据库(可选)

如果你的数据量很大，可以使用自定义的数据库，否则的话，可能会出现一些错误，比如当主题数超过10000的时候无法显示主题的信息。 使用 PostgreSQL 的示例如下：

  1. 使用这个文件初始化数据库以及表结构。
  2. 下载并修改 配置文件，然后添加PostgreSQL的配置。
spring.datasource.driver-class-name=org.postgresql.Driver
spring.datasource.url=jdbc:postgresql://127.0.0.1:5432/pulsar_manager
spring.datasource.username=postgres
spring.datasource.password=postgres
  1. 添加配置挂载并启动一个docker镜像
docker pull apachepulsar/pulsar-manager:v0.2.0
docker run -it \
    -p 9527:9527 -p 7750:7750 \
    -v /your-path/application.properties:/pulsar-manager/pulsar-
manager/application.properties
    -e SPRING_CONFIGURATION_FILE=/pulsar-manager/pulsar-manager/application.properties \
    apachepulsar/pulsar-manager:v0.2.0

开启JWT身份认证(可选)

如果你想打开JWT身份认证的话，配置application.properties 如下：

backend.jwt.token=token

jwt.broker.token.mode=PRIVATE
jwt.broker.public.key=file:///path/broker-public.key
jwt.broker.private.key=file:///path/broker-private.key

or 
jwt.broker.token.mode=SECRET
jwt.broker.secret.key=file:///path/broker-secret.key

backend.jwt.token: 超级用户的token 需要在集群初始化时配置此参数。 jwt.broker.token.mode:多种生成token的模式，包括PUBLIC, PRIVATE, 和SECRET。 jwt.broker.public.key：如果你使用PUBLIC模式需要配置这个选项。 jwt.broker.private.key：如果你使用PRIVATE模式需要配置这个选项。 jwt.broker.secret.key：如果你使用SECRET模式需要配置这个选项。 更多信息，请查看Token Authentication Admin of Pulsar。

添加配置以及秘钥的docker命令如下：

docker pull apachepulsar/pulsar-manager:v0.2.0
docker run -it \
    -p 9527:9527 -p 7750:7750 \
    -v /your-path/application.properties:/pulsar-manager/pulsar-
manager/application.properties
    -v /your-path/private.key:/pulsar-manager/private.key
    -e SPRING_CONFIGURATION_FILE=/pulsar-manager/pulsar-manager/application.properties \
    apachepulsar/pulsar-manager:v0.2.0

设置管理员账号以及密码

CSRF_TOKEN=$(curl http://localhost:7750/pulsar-manager/csrf-token)
curl \
   -H 'X-XSRF-TOKEN: $CSRF_TOKEN' \
   -H 'Cookie: XSRF-TOKEN=$CSRF_TOKEN;' \
   -H "Content-Type: application/json" \
   -X PUT http://localhost:7750/pulsar-manager/users/superuser \
   -d '{"name": "admin", "password": "apachepulsar", "description": "test", "email": "username@test.org"}'

Curl命令中的请求参数：

{"name": "admin", "password": "apachepulsar", "description": "test", "email": "username@test.org"}
  • name是Pulsar Manger的登录用户名，当前是admin。
  • password 是Pulsar Manager当前用户的密码，当前是apachepulsar。 密码应该大于等于6位。

配置环境

  1. 登录系统，访问http://localhost:9527去登录。 当前的默认账号是 admin/apachepulsar

  2. 点击"New Environment"去添加一个环境。

  3. 输入"Environment Name"。 环境名称是用来识别确定一个环境的。

  4. 输入"Service URL"。 服务URL是你Pulsar集群的管理服务URL。

其他安装方式

裸金属安装

当你使用二进制包的方式直接部署的时候，你可以参考以下步骤。

  • 下载并解压二进制包，二进制包可在 Pulsar Download上下载。

      wget https://dist.apache.org/repos/dist/release/pulsar/pulsar-manager/pulsar-manager-0.2.0/apache-pulsar-manager-0.2.0-bin.tar.gz
  tar -zxvf apache-pulsar-manager-0.2.0-bin.tar.gz

  • 解压二进制服务包，并将前端资源放在后端服务的目录中。

      cd pulsar-manager
  tar -zxvf pulsar-manager.tar
  cd pulsar-manager
  cp -r ../dist ui

  • 按需修改 application.properties配置

    如果你不想修改 application.properties文件，你可以添加配置到启动参数。 /bin/pulsar-manager --backend.jwttoken=token 这完全兼容spring boot框架。

  • 启动Pulsar Manger

    ./bin/pulsar-manager

自定义dokcer镜像安装

你能在 Docker Hub 找到这个 docker 镜像，这个镜像是基于下面的源码构建的：

git clone https://github.com/apache/pulsar-manager
cd pulsar-manager/front-end
npm install --save
npm run build:prod
cd ..
./gradlew build -x test
cd ..
docker build -f docker/Dockerfile --build-arg BUILD_DATE=`date -u +"%Y-%m-%dT%H:%M:%SZ"` --build-arg VCS_REF=`latest` --build-arg VERSION=`latest` -t apachepulsar/pulsar-manager .
