如何连接 Pulsar 到数据库

本教程提供了上手机会，让你了解如何在不写一行代码的情况下将数据移出 Pulsar。

运行本指南中的步骤让你在阅读 Pulsar I/O 概念时会有更深入的理解。

本教程结束后，你可以：

  • 连接 Pulsar 到 Cassandra

  • 连接 Pulsar 到 PostgreSQL

提示

  • 本教程假定你以单机模式运行 Pulsar。 但是，本教程中使用的所有命令无需做任何更改也可在多节点 Pulsar 集群中使用。

  • 所有操作指南假定是在 Pulsar 二进制文件的根目录下运行。

安装 Pulsar 和内置连接器

在连接 Pulsar 到数据库之前，你需要安装 Pulsar 和所需的内置连接器。

关于如何安装单机 Pulsar 和内置连接器的更多信息，请参阅这里。

启动单机模式 Pulsar

  1. 本地启动 Pulsar。

    bin/pulsar standalone

    按顺序启动 Pulsar 服务的所有组件。

    你可以利用 curl 命令向这些 Pulsar 服务端点发送请求，确保 Pulsar 服务运行正常。

  2. 检查 Pulsar 二进制协议端口。

    telnet localhost 6650

  3. 检查 Pulsar Function 集群。

    curl -s http://localhost:8080/admin/v2/worker/cluster

    示例输出

    [{"workerId":"c-standalone-fw-localhost-6750","workerHostname":"localhost","port":6750}]

  4. 请确保公共租户和默认命名空间存在。

    curl -s http://localhost:8080/admin/v2/namespaces/public

    示例输出

    ["public/default","public/functions"]

  5. 所有内置连接器都应列为 available。

    curl -s http://localhost:8080/admin/v2/functions/connectors

    示例输出

    [{"name":"aerospike","description":"Aerospike database sink","sinkClass":"org.apache.pulsar.io.aerospike.AerospikeStringSink"},{"name":"cassandra","description":"Writes data into Cassandra","sinkClass":"org.apache.pulsar.io.cassandra.CassandraStringSink"},{"name":"kafka","description":"Kafka source and sink connector","sourceClass":"org.apache.pulsar.io.kafka.KafkaStringSource","sinkClass":"org.apache.pulsar.io.kafka.KafkaBytesSink"},{"name":"kinesis","description":"Kinesis sink connector","sinkClass":"org.apache.pulsar.io.kinesis.KinesisSink"},{"name":"rabbitmq","description":"RabbitMQ source connector","sourceClass":"org.apache.pulsar.io.rabbitmq.RabbitMQSource"},{"name":"twitter","description":"Ingest data from Twitter firehose","sourceClass":"org.apache.pulsar.io.twitter.TwitterFireHose"}]

    如果在启动 Pulsar 服务时出现错误，你可能会在运行 pulsar/standalone 的终端发现异常， 或者你可以到 Pulsar 目录下的 logs 目录来查看日志。

连接 Pulsar 到 Cassandra

本节展示如何连接 Pulsar 到 Cassandra。

提示

  • 请确保你已经安装了 Docker。如果还没安装，详见安装 Docker。

  • Cassandra sink 连接器从 Pulsar 主题读取消息，并将消息写入 Cassandra 表。更多详细信息，请参阅 Cassandra sink connector。

配置 Cassandra 集群

本示例使用 cassandra Docker 镜像在 Docker 中启动单节点 Cassandra 集群。

  1. 启动 Cassandra 集群。

    docker run -d --rm --name=cassandra -p 9042:9042 cassandra

    备注

    进行下一个步骤前，请确保 Cassandra 集群正在运行。

  2. 确保 Docker 进程正在运行。

    docker ps

  3. 检查 Cassandra 日志以确保 Cassandra 进程按预期运行。

    docker logs cassandra

  4. 检查 Cassandra 集群的状态。

    docker exec cassandra nodetool status

    示例输出

     Datacenter: datacenter1
 =======================
 Status=Up/Down
 |/ State=Normal/Leaving/Joining/Moving
 --  Address     Load       Tokens       Owns (effective)  Host ID                               Rack
 UN  172.17.0.2  103.67 KiB  256          100.0%            af0e4b2f-84e0-4f0b-bb14-bd5f9070ff26  rack1

  5. 使用 cqlsh 连接到 Cassandra 集群。

    $ docker exec -ti cassandra cqlsh localhost
Connected to Test Cluster at localhost:9042.
[cqlsh 5.0.1 | Cassandra 3.11.2 | CQL spec 3.4.4 | Native protocol v4]
Use HELP for help.
cqlsh>

  6. 创建键空间 pulsar_test_keyspace。

    cqlsh> CREATE KEYSPACE pulsar_test_keyspace WITH replication = {'class':'SimpleStrategy', 'replication_factor':1};

  7. 创建表 pulsar_test_table。

    cqlsh> USE pulsar_test_keyspace;
cqlsh:pulsar_test_keyspace> CREATE TABLE pulsar_test_table (key text PRIMARY KEY, col text);

配置 Cassandra sink

现在我们有一个本地运行的 Cassandra 集群。

在本节中，你需要配置一个 Cassandra sink 连接器。

要运行 Cassandra sink 连接器，你需要准备一个配置文件，其中包含 Pulsar 连接器运行时需要知道的信息。

例如，Pulsar 连接器如何找到 Cassandra 集群， Pulsar 连接器用于写入消息的键空间和表，等等。

你可以通过以下任意一种方法创建配置文件。

  • JSON

    {
    "roots": "localhost:9042",
    "keyspace": "pulsar_test_keyspace",
    "columnFamily": "pulsar_test_table",
    "keyname": "key",
    "columnName": "col"
}

  • YAML

    configs:
    roots: "localhost:9042"
    keyspace: "pulsar_test_keyspace"
    columnFamily: "pulsar_test_table"
    keyname: "key"
    columnName: "col"

更多详细信息，请参阅 Cassandra sink connector。

创建 Cassandra sink

你可以使用连接器管理 CLI 来创建 sink 连接器并执行其他操作。

结合先前创建的 examples/cassandra-sink.yml 配置文件，运行下面的命令创建 sink 类型为 cassandra 的 Cassandra sink 连接器。

备注

当前内置连接器的sink-type参数由 pulsar-io.yaml 文件中指定的name参数设置决定。

bin/pulsar-admin sinks create \
    --tenant public \
    --namespace default \
    --name cassandra-test-sink \
    --sink-type cassandra \
    --sink-config-file examples/cassandra-sink.yml \
    --inputs test_cassandra

执行命令后，Pulsar 会创建 sink 连接器 cassandra-test-sink。

此 sink 连接器作为 Pulsar Function 运行，并将主题 test_cassandra 产生的消息写入 Cassandra 表 pulsar_test_table 中。

连接 Pulsar 到 PostgreSQL

本节展示如何连接 Pulsar 到 PostgreSQL。

提示

  • 请确保你已经安装了 Docker。如果还没安装，详见安装 Docker。

  • JDBC sink 连接器从 Pulsar 主题拉取消息，并将消息持久化到 ClickHouse、MariaDB、PostgreSQL 或 SQlite。 更多详细信息，请参阅 JDBC sink connector。

配置 PostgreSQL 集群

本示例使用 PostgreSQL 12 Docker 镜像在 Docker 中启动单节点 PostgreSQL 集群。

  1. 从 Docker 拉取 PostgreSQL 12 镜像。

    $ docker pull postgres:12

  2. 启动 PostgreSQL。

    $ docker run -d -it --rm \
--name pulsar-postgres \
-p 5432:5432 \
-e POSTGRES_PASSWORD=password \
-e POSTGRES_USER=postgres \    
postgres:12

    提示

    参数说明示例
    -d以分离模式启动容器。/
    -it即使没有连接，也要保持 STDIN 开启并分配终端。/
    --rm容器退出时会自动删除。/
    -name为容器指定名称。此示例为容器指定 pulsar-postgres。
    -p将容器的端口发布到主机。此示例将容器的端口 5432 发布到主机。
    -e设置环境变量此示例设置以下变量：
    • 用户密码 password。
    • 用户名称 postgres。 |

    提示

    关于 Docker 命令的更多信息，详见 Docker CLI。

  3. 检查 PostgreSQL 是否已成功启动。

    $ docker logs -f pulsar-postgres

    如果出现以下消息说明 PostgreSQL 已成功启动。

    2020-05-11 20:09:24.492 UTC [1] LOG:  starting PostgreSQL 12.2 (Debian 12.2-2.pgdg100+1) on x86_64-pc-linux-gnu, compiled by gcc (Debian 8.3.0-6) 8.3.0, 64-bit
2020-05-11 20:09:24.492 UTC [1] LOG:  listening on IPv4 address "0.0.0.0", port 5432
2020-05-11 20:09:24.492 UTC [1] LOG:  listening on IPv6 address "::", port 5432
2020-05-11 20:09:24.499 UTC [1] LOG:  listening on Unix socket "/var/run/postgresql/.s.PGSQL.5432"

2020-05-11 20:09:24.533 UTC [1] LOG:  database system is ready to accept connections

  4. 访问 PostgreSQL。

    $ docker exec -it pulsar-postgres /bin/bash

  5. 创建 PostgreSQL 表 pulsar_postgres_jdbc_sink。

    $ psql -U postgres postgres

postgres=# create table if not exists pulsar_postgres_jdbc_sink
(
id serial PRIMARY KEY,

);

配置 JDBC sink

现在我们有一个在本地运行的 PostgreSQL 。

在本节中，你需要配置 JDBC sink 连接器。

  1. 添加配置文件。

    要运行 JDBC sink 连接器，你需要准备一个 YAML 配置文件，其中包含 Pulsar 连接器运行时需要知道的信息。

    例如，Pulsar 连接器如何找到 PostgreSQL 集群，JDBC URL 是什么以及 Pulsar 连接器用于写入消息的表。

    创建 pulsar-postgres-jdbc-sink.yaml 文件，将以下内容复制到文件中，并将该文件放在 pulsar/connectors 文件夹中。

    configs:
  userName: "postgres"
  password: "password"
  jdbcUrl: "jdbc:postgresql://localhost:5432/postgres"
  tableName: "pulsar_postgres_jdbc_sink"

  2. 创建 Schema。

    创建 avro-schema 文件，将以下内容复制到该文件，并将该文件放在 pulsar/connectors 文件夹中。

    {
  "type": "AVRO",
  "schema": "{\"type\":\"record\",\"name\":\"Test\",\"fields\":[{\"name\":\"id\",\"type\":[\"null\",\"int\"]},{\"name\":\"name\",\"type\":[\"null\",\"string\"]}]}",
  "properties": {}
}

    提示

    关于 AVRO 的更多信息，详见 Apache Avro。

  3. 将 Schema 上传到主题。

    此示例将 avro-schema schema 上传到 pulsar-postgres-jdbc-sink-topic 主题中。

    $ bin/pulsar-admin schemas upload pulsar-postgres-jdbc-sink-topic -f ./connectors/avro-schema

  4. 检查 Schema 是否成功上传。

    $ bin/pulsar-admin schemas get pulsar-postgres-jdbc-sink-topic

    如果出现以下消息，则说明 Schema 上传成功。

    {"name":"pulsar-postgres-jdbc-sink-topic","schema":"{\"type\":\"record\",\"name\":\"Test\",\"fields\":[{\"name\":\"id\",\"type\":[\"null\",\"int\"]},{\"name\":\"name\",\"type\":[\"null\",\"string\"]}]}","type":"AVRO","properties":{}}

创建 JDBC sink

你可以使用连接器管理 CLI 来创建 sink 连接器并执行其他操作。

此示例展示创建 sink 连接器和指定想要的信息。

$ bin/pulsar-admin sinks create \
--archive ./connectors/pulsar-io-jdbc-postgres-2.9.1.nar \
--inputs pulsar-postgres-jdbc-sink-topic \
--name pulsar-postgres-jdbc-sink \
--sink-config-file ./connectors/pulsar-postgres-jdbc-sink.yaml \
--parallelism 1

执行命令后，Pulsar 会创建 sink 连接器 pulsar-postgres-jdbc-sink。

此 sink 连接器作为 Pulsar Function 运行，并将主题 pulsar-postgres-jdbc-sink-topic 产生的消息写入 PostgreSQL 表 pulsar_postgres_jdbc_sink 中。

提示

参数说明示例
--archiveSink 归档文件的路径。pulsar-io-jdbc-postgres-2.9.1.nar
--inputsSink 的输入主题。

多个主题可以指定为逗号分隔的列表。 | | | --name | Sink 的名称 | pulsar-postgres-jdbc-sink | | --sink-config-file | 指定 sink 配置的 YAML 配置文件的路径。 | pulsar-postgres-jdbc-sink.yaml | | --parallelism | Sink 的并行度系数。

比如要运行的 sink 实例的数量。 | 1 |

提示

更多关于 pulsar-admin sinks create options 的信息，参阅这里。

如果出现以下消息，则说明 sink 创建成功。

Created successfully

停止 JDBC sink

$ bin/pulsar-admin sinks stop \
--tenant public \
--namespace default \
--name pulsar-postgres-jdbc-sink

提示

更多关于 pulsar-admin sinks stop options 的信息，参阅这里。

Stopped successfully

$ bin/pulsar-admin sinks restart \
--tenant public \

提示
