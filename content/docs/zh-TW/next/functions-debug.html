Pulsar Function 除錯

可以使用以下方法來對 Pulsar Function 進行除錯：

  • 捕捉 stderr
  • 使用單元測試
  • 本地運行模式除錯
  • 使用日誌 topic
  • 使用 Function CLI

捕捉 stderr

將 Function 啟動訊息和捕捉到的 stderr 輸出寫到 logs/functions/<tenant>/<namespace>/<function>/<function>-<instance>.log 日誌檔。

這對於 Function 為什麼無法啟動的除錯很有用。

使用單元測試

一個 Pulsar Function 是具有輸入和輸出的函數，你可以像測試其它函數一樣的手法來對 Pulsar Function 進行測試。

例如，如果你有以下 Pulsar Function：

import java.util.function.Function;

public class JavaNativeExclamationFunction implements Function<String, String> {
   @Override
   public String apply(String input) {
       return String.format("%s!", input);
   }
}

可以編寫一個簡單的單元測試來測試 Pulsar Function。

提示

Pulsar uses testng for testing.

@Test
public void testJavaNativeExclamationFunction() {
   JavaNativeExclamationFunction exclamation = new JavaNativeExclamationFunction();
   String output = exclamation.apply("foo");
   Assert.assertEquals(output, "foo!");
}

以下 Pulsar Function 實現了 org.apache.pulsar.functions.api.Function 介面接口。

import org.apache.pulsar.functions.api.Context;
import org.apache.pulsar.functions.api.Function;

public class ExclamationFunction implements Function<String, String> {
   @Override
   public String process(String input, Context context) {
       return String.format("%s!", input);
   }
}

在這種情況下，你也可以為此 function 編寫單元測試。 記住要模擬 Context 參數。 以下是一個範例。

提示

Pulsar uses testng for testing.

@Test
public void testExclamationFunction() {
   ExclamationFunction exclamation = new ExclamationFunction();
   String output = exclamation.process("foo", mock(Context.class));
   Assert.assertEquals(output, "foo!");
}

本地運行模式除錯

當在 localrun 模式下運行 Pulsar Function 時，它將使用線程在本機上啟動 Function 的實例。

在此模式下，Pulsar Function 將消費與生成數據到 Pulsar 集群，並模擬該 function 在 Pulsar集群中實際運行的方式。

注意
當前僅 Java 編寫的 Pulsar Functions 支持使用 localrun 模式進行除錯。 你需要 Pulsar 2.4.0 或更高版本才能執行以下操作。 即使在 Pulsar 2.4.0 之前的版本中都可以使用 localrun，但是這些舊版本的 Pulsar 不能以 localrun 模式進行編程調試或作為線程運行。

你可以通過以下方式啟動 function。

FunctionConfig functionConfig = new FunctionConfig();
functionConfig.setName(functionName);
functionConfig.setInputs(Collections.singleton(sourceTopic));
functionConfig.setClassName(ExclamationFunction.class.getName());
functionConfig.setRuntime(FunctionConfig.Runtime.JAVA);
functionConfig.setOutput(sinkTopic);

LocalRunner localRunner = LocalRunner.builder().functionConfig(functionConfig).build();
localRunner.start(true);

透過這個方法就可以使用 IDE 來對 Pulsar Function 來進行除錯。 設置斷點並手動逐步地執行 function 來以實際數據進行除錯。

以下範例說明如何以編程方式在 localrun 模式下啟動 function。

public class ExclamationFunction implements Function<String, String> {

   @Override
   public String process(String s, Context context) throws Exception {
       return s + "!";
   }

public static void main(String[] args) throws Exception {
    FunctionConfig functionConfig = new FunctionConfig();
    functionConfig.setName("exclamation");
    functionConfig.setInputs(Collections.singleton("input"));
    functionConfig.setClassName(ExclamationFunction.class.getName());
    functionConfig.setRuntime(FunctionConfig.Runtime.JAVA);
    functionConfig.setOutput("output");

要以編程方式使用 localrun 模式，需要添加以下依賴項。

<dependency>
   <groupId>org.apache.pulsar</groupId>
   <artifactId>pulsar-functions-local-runner</artifactId>
   <version>${pulsar.version}</version>
</dependency>

有關完整的代碼，請參見 here。

注意
不久之後將支持使用 localrun 模式調試以其他語言編寫的Pulsar Function。

使用日誌 topic

在 Pulsar Function 中，可以產生 function 中定義的日誌信息，並將日誌拋送至指定日誌 topic。 你可以配置 consumer 消費指定的日誌 topic 來檢查日誌訊息。

範例

import org.apache.pulsar.functions.api.Context;
import org.apache.pulsar.functions.api.Function;
import org.slf4j.Logger;

public class LoggingFunction implements Function<String, Void> {
    @Override
    public void apply(String input, Context context) {
        Logger LOG = context.getLogger();
        String messageId = new String(context.getMessageId());

        if (input.contains("danger")) {
            LOG.warn("A warning was received in message {}", messageId);
        } else {
            LOG.info("Message {} received\nContent: {}", messageId, input);
        }

        return null;
    }
}

如上面的範例所示，可以通過 context.getLogger() 來獲取 logger 物件實例，並將 logger 分配給 slf4j 的 LOG 變數，因此可以使用 LOG 變數在 function 中去拋出所需的日誌訊息。 同時，需要指定產生日誌訊息的 topic。

範例

$ bin/pulsar-admin functions create \
  --log-topic persistent://public/default/logging-function-logs \
  # Other function configs

The message published to log topic contains several properties for better reasoning:

  • loglevel -- the level of the log message.
  • fqn -- fully qualified function name pushes this log message.
  • instance -- the ID of the function instance pushes this log message.

使用 Function CLI

With Pulsar Functions CLI, you can debug Pulsar Functions with the following subcommands:

  • get
  • status
  • stats
  • list
  • trigger

提示

For complete commands of Pulsar Functions CLI, see here。

get

Get information about a Pulsar Function.

Usage

$ pulsar-admin functions get options

Options

旗標說明
--fqfnPulsar Function 的 Fully Qualified Function Name (FQFN)
--namePulsar Function 的名稱
--namespacePulsar Function 的命名空間
--tenantPulsar Function 的租戶名稱

提示

--fqfn 包含了 --name、--namespace 與 --tenant, 因此你可以指定--fqfn 或 --name、--namespace 與 --tenant。

Example

You can specify --fqfn to get information about a Pulsar Function.

$ ./bin/pulsar-admin functions get public/default/ExclamationFunctio6

Optionally, you can specify --name, --namespace and --tenant to get information about a Pulsar Function.

$ ./bin/pulsar-admin functions get \
    --tenant public \
    --namespace default \
    --name ExclamationFunctio6

As shown below, the get command shows input, output, runtime, and other information about the ExclamationFunctio6 function.

trigger

Trigger a specified Pulsar Function with a supplied value. This command simulates the execution process of a Pulsar Function and verifies it.

Usage

$ pulsar-admin functions trigger options

Options

旗標說明
--fqfnPulsar Function 的 Fully Qualified Function Name (FQFN)
--namePulsar Function 的名稱
--namespacePulsar Function 的命名空間
--tenantPulsar Function 的租戶名稱
--topicPulsar Function 要消費的 topic 名稱
--trigger-file包含觸發 Pulsar Function 的數據文件路徑。
--trigger-value觸發 Pulsar Function 的值

Example

$ ./bin/pulsar-admin functions trigger \
    --tenant public \
    --namespace default \
    --name ExclamationFunctio6 \
    --topic persistent://public/default/my-topic-1 \
    --trigger-value "hello pulsar functions"

As shown below, the trigger command returns the following result:

This is my function!

Note

You must specify the entire topic name when using the --topic option. 否則，將發生以下錯誤。

Function in trigger function has unidentified topic
